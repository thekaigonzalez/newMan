manual.css(1) - Manual styling for HTML

Author: Kai D. Gonzalez (gkai70264@gmail.com)

classes

            this css document contains classes which are used to
            emulate the styles of manual page sites with a spin on
            modern styling sheets.

            Classes
                .document { ... } /* the basic outline of the manual document */                              
                
                .highlight { ... } /* changes the color to a more dark orange-red. 
                                      This class is the default for <b> tags */

                .parameter { ... } /* changes the text to a more blu-ish color. */
                
                ._title { ... } /* giant green title at the top of the document. */ 

                .author { ... } /* creates a foggy, inlined text for the
                                   start of the document to give a nice structured 
                                   author introduction. */
                
                .subheader { ... } /* an uppercase header. */
            
            these styles work off of pre tags and do not wrap inside 
            of the main document. However you format text is how it will end
            up looking on paper. the main document contains lower width and
            rounded corners, with a slight shadow in the back, with 0.1 
            opacity.

            As you can see to the right, the image is floated to that direction
            to give the right sided effect of a typical newsletter, which many
            other online manual pages do not support, this one does. The images
            also contain animations when you hover over them, using a smooth API
            they can easily be customized and edited with the touch of a few
            keys.

            the styling is heavily inspired by lua.org, the main website styles.
            With a hint of man7.org, manpage.io, and linux.die.net
            but edited to support a bit more scalable formatting and a modern
            spin on old documentation.

supported tags

            The supported tags are as follows:

                    Regular Highlight
                    Parameter
                    Italic Green
                    subheader (forces uppercase)
                    Link [example.com]
                    Regular Images (see above)
                    Author (forces lowercase) - Lightly credited
